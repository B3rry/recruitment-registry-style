Recruitment Registry Style Guide

Last updated on October 31st, 2016

Colors

  • Display Box sould be the only components with non explicit color variables. They utilize background-color: lighten($, 33%); to calculate the lighter background fill.

Typography

  • Typefaces are separated into three different styles for different use cases. (There is an additional monospace font declaration that is unusued).
  • By default, text is styled as Raleway by using the $font-primary SASS variable. This is used for all general use text.
  • Tooltip text, flavor text, and other informational text uses $font-secondary, which is Lato. Apply .label for instances of this text as a label. Apply .tooltip for instances of this text as a tooltip.
  • Questions and other similar components use Argent CF bold. This is called via $font-serif. For question text, please apply .question to a p tag with your content.

Scaffolding

  • Scaffolding is implemented via flexboxgrid.
  • Documentation is available at: http://flexboxgrid.com
  • Consider using the sass property extend to reduce the number of utility classes exposed in the html. Documentation.

Animation

  • Animation is implemented via animate.css.
  • Documentation is available at: https://github.com/daneden/animate.css
  • Consider using the sass property extend to reduce the number of utility classes exposed in the html. Documentation.

Buttons

  • Buttons that preform an action should utilize the button tag.
  • Buttons that take you to a new location should utilize the a tag. NOT EVERY a TAG NEEDS TO BE A BUTTON, THERE ARE MULTIPLE INSTANCES WHERE THEY STAY AS DEFAULT A TAG STYLING IN THE COMPS
  • Buttons should always include either the .buttonPrimary or .buttonSecondary class.
  • Additionally, you may need to specify a specific style of button. Use the following classes to do so: .light .success .confirm .alert .warning .info .system .dark

Select

  • Dropdowns are styled by using the default select implementation. W3 Schools - HTML select Tag documentation
  • Selects use a custom background-image for dropdown indicator custom style. This reuqires a SASS function to execute. If you must change the color outside of the normal classes, this color must be passed as a HEX value. See the implementations in ./client/src/styles/components/_form.scss
  • You may need to specify a specific style of select. Use the following classes to do so: .light .success .confirm .alert .warning .info .system .dark

Radio Groups

  • Radio groups are styled by using the default radio implementation. W3 Schools - HTML Form Radio Button input
  • You may need to specify a specific style of radio group. Use the following classes to do so: .light .success .confirm .alert .warning .info .system .dark

Display Boxes (Alerts/Notifications)

  • Display boxes are used to show alerts and notifications to a user. This includes validation messages and warnings. Use .displayBox to extend the class to your div
  • Consider adding .animated and .fadeInUp to notifications that appear to a user after an action.
  • This is the only section where colors are not defined as explicit variables. Use background-color: lighten($, 33%); when constructing new displayBox types.
  • You may need to specify a specific style of display box. Use the following classes to do so: .light .success .confirm .alert .warning .info .system .dark

Utility & UI Classes

  • Consider using the sass property extend to reduce the number of utility classes exposed in the html. Documentation.
  • The class .pull-right is available for floating elements to the right.
  • The classes .utility--shadow-1 to .utility--shadow-5 are avaialable to set shadow levels, matching those from Google's material design guidelines.
  • .utility--shadow-1 includes a hover state by default. To remove the hover state, extend with the class .noRaise
  • The class .utility--shadow-flat is available for when you want to have a shadow on :hover only. Additionally, you can extend with .small to create a smaller shadow on hover state. This class can also be extended with .active or .activeS to create :active state hover effects as well. (Great for buttons that need more emphasis).
